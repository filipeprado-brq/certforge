| app → localStorage | Theme preference and (later) study state written through Plan 01's storage layer. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05 | Repudiation/Destructive action | Settings "Reset all progress" | mitigate | Destructive reset is gated behind a ConfirmDialog (alertdialog, explicit "Yes, reset everything" label) so it cannot fire on a single accidental click |
| T-01-06 | Tampering | theme value driving `data-theme` | mitigate | theme resolves to a fixed union ('light'|'dark'); any stored themePref outside 'system'|'light'|'dark' falls back to defaults via Plan 01 readState validation |
| T-01-07 | Information Disclosure | rendered study state | accept | All state is the user's own non-sensitive study progress on their device; no PII, no network egress (offline-first per PROJECT.md) |
</threat_model>
